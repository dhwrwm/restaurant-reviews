# Learning Notes

This section is different in register from the rest of the handbook. The
other sections ([Architecture](/architecture), [Features](/features),
[ADR](/adr)) describe the system as it is or will be, written for anyone
onboarding onto the project. Learning Notes are personal, informal, and
in-progress — things learned, discovered, or found surprising while building
each part of the stack. They're allowed to be incomplete, first-person, and
opinionated in a way the rest of the handbook isn't.

Pages here accumulate as each topic is actually worked on — most are short
today because most of the deeper phases (Go, Redis, AI) haven't started
yet. See [Roadmap](/roadmap) for when each topic is expected to have more to
say.
