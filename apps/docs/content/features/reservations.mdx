# Reservations

<Callout type="warning">**Status:** 📋 Planned — no booking model, availability logic, or UI exists today.</Callout>

## Overview

Would let a reviewer request/hold a table at a restaurant for a given time.

## Purpose

Not scoped in detail yet — this is the most speculative feature on the
roadmap. It's listed here because it's a plausible platform extension, not
because there's a committed design.

## Planned shape

Genuinely open. At minimum it implies a `Reservation` model
(`restaurantId`, `reviewerId`, time slot, status) and some notion of
capacity/availability per restaurant, which doesn't exist in the schema
today. Whether availability is owner-configured or a fixed capacity number
is undecided.

## Trade-offs

Not yet applicable — nothing is built, and the design isn't settled enough
to evaluate trade-offs.
