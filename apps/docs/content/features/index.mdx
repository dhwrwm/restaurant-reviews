# Features

Each page below follows the same structure: Overview, Status, Purpose,
Architecture, Implementation, Trade-offs, Future Improvements, References —
plus Database/API/Frontend/Backend/Security/Performance subsections where
they apply. A 📋 Planned feature doesn't have a real Backend or Database
section to document yet, so it gets a short "planned shape" note instead.

| Feature | Status | Notes |
| --- | --- | --- |
| [Authentication](/features/authentication) | ✅ Implemented | JWT access/refresh, Argon2, HTTP-only cookies |

| [Restaurants](/features/restaurants) | ✅ Implemented | Owner CRUD, filterable/paginated listing |
