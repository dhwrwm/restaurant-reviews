## Local development

```bash
pnpm db:migrate   # prisma migrate dev — creates + applies a migration
pnpm db:generate  # prisma generate — regenerate the Prisma client only
```

## Production (Render free tier)

Render's Pre-Deploy Command feature — the natural place to run
`prisma migrate deploy` — isn't available on the free plan. Migrations
instead run at the front of the API's `startCommand`, immediately before
`node dist/main`:

```bash
pnpm --filter api exec prisma migrate deploy && pnpm --filter api start:prod
```

`prisma migrate deploy` is a no-op when nothing is pending, so this is safe
on every boot — including the free tier's spin-down/wake cycle, which needs
to pick up any new migration anyway. See
[Infrastructure: Deployment](/infrastructure/deployment).
