# Restaurant Review Platform

Engineering handbook for the **Restaurant Review Platform** — a full-stack
application where reviewers discover, rate, and review restaurants, and
restaurant owners manage their own listings.

This is documentation written the way an engineering team would write it for
itself: what's actually built, what's in progress, what's planned, and why.
Every page carries a status —**✅ Implemented**, **🚧 In Progress**, or
**📋 Planned** — so nothing here is mistaken for a feature that exists when it
doesn't.

## Start here

- New to the project? Start with [Introduction](/introduction) and
  [Local Development](/getting-started).
- Want the system design? Go to [Architecture](/architecture).
- Looking for a specific feature's implementation? See [Features](/features).
- Need the schema or an endpoint contract? See [Database](/database) or
  [API](/api).
- Curious where this is headed? See [Roadmap](/roadmap) and
  [Architecture Decision Records](/adr).
