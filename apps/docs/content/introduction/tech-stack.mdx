# Tech Stack

## Implemented today

| Layer | Technology |
| --- | --- |
| Frontend | Next.js (App Router), React, TypeScript, Tailwind CSS |
| Backend | NestJS, Prisma ORM |
| Database | PostgreSQL (hosted on [Neon](https://neon.tech)) |
| Auth | JWT (access + refresh), Argon2 password hashing |
| Monorepo | Turborepo, pnpm workspaces |
| Hosting | Vercel (web + docs), Render (API) |
| Docs | Next.js + Nextra |

This list is intentionally short — it's everything that appears in a
dependency file or a deploy config today. See
[Repository Structure](/introduction/repository-structure) for where each
piece lives in the monorepo.

## Infrastructure philosophy

Technology choices, current and future, are made to keep the project
reproducible on free tiers, open-source where practical, and easy to swap out:

- Prefer generous free tiers (Vercel, Render, Neon, Upstash) over paid
  defaults.
- Prefer open-source, Docker-compatible technology over managed-only
  services, so any piece can be self-hosted later without a rewrite.
- Avoid vendor lock-in — no component is chosen such that leaving the vendor
  means rearchitecting.

Paid services (e.g. Google Places instead of OpenStreetMap + Leaflet) are
only ever documented as optional alternatives, never as the default.
