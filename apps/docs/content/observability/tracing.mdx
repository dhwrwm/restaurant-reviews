# Tracing

<Callout type="warning">**Status:** 📋 Planned — no tracing instrumentation exists today.</Callout>

## Purpose

Tracing matters most once a request can span multiple services — today a
request touches exactly one of `apps/api` or `apps/web`, then Postgres, so
there's no cross-service path to trace yet. This becomes genuinely useful
once [Go services](/roadmap/phase-3-redis-go-services) exist and a single
user-facing request might fan out across NestJS, a Go search service, and
Redis.

## Planned shape

- OpenTelemetry tracing, introduced alongside (or shortly after) the first
  extracted Go service — see
  [Phase 3](/roadmap/phase-3-redis-go-services) and
  [Phase 6](/roadmap/phase-6-observability).
- Deferred ahead of that because a single-service request has little to
  gain from a trace over a well-correlated log line — see
  [Logging](/observability/logging).
