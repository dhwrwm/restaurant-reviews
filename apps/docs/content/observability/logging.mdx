# Logging

<Callout type="warning">**Status:** 📋 Planned — `apps/api` uses NestJS's default console logger; there's no structured log format or aggregation today.</Callout>

## Purpose

Console logs are sufficient for local development and for reading Render's
log stream directly, but aren't structured (JSON) or aggregated anywhere —
searching across a time range or correlating logs to a specific request
isn't currently possible.

## Planned shape

- Structured (JSON) logging with a request ID on every log line, to allow
  correlating a single request's logs together — this is also a
  prerequisite for [Tracing](/observability/tracing).
- Shipping logs to a log aggregation service once volume justifies it (a
  free-tier option, per this project's
  [infrastructure philosophy](/introduction/tech-stack#infrastructure-philosophy),
  would be evaluated first).
